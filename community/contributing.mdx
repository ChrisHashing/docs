# Contributing to WTTP

We’re excited to have you join the WTTP development journey! WTTP is an open-source project built to decentralize web protocols and empower Web3 developers. Your contributions can help us create a more resilient, efficient, and decentralized internet.

---

## **How to Contribute**

### 1. **Fork the Repository**
Start by forking the WTTP GitHub repository:

[WTTP GitHub Repository](https://github.com/TechnicallyWeb3/wttp)

Click the **Fork** button in the top-right corner to create a personal copy of the repository.

---

### 2. **Clone the Repository**
Clone your forked repository to your local machine:

```bash
git clone https://github.com/<your-username>/wttp.git
```

Navigate to the project directory:

```bash 
cd wttp
```

### 3. **Set Up the Environment**

Follow these steps to set up the project locally:

Install dependencies:

```bash 
npm install
```

Build the project:

```bash 
npm run build
```
### 4. **Pick an Issue**

Browse the list of open issues on the Issues Page. Look for issues labeled good first issue or help wanted to get started.

If you have an idea for an improvement or find a bug, feel free to open a new issue and describe it in detail.

### 5. **Create a New Branch**

Before making any changes, create a new branch for your work:
```bash 
git checkout -b feature/your-feature-name
```
Replace your-feature-name with a descriptive name for your changes.

### 6. **Make Changes**

Make your changes to the codebase. Ensure your code follows the project’s coding standards:

- Adhere to the existing code style.
- Write clear and concise comments where necessary.
- Add or update tests for your changes.

### 7. **Test Your Changes**

Before submitting your changes, make sure all tests pass:

```bash 
npm run test
```
If you’ve added new functionality, consider writing additional tests.

### 8. **Commit Your Changes**

Commit your changes with a descriptive commit message:

```bash 
git add .
git commit -m "Add detailed description of your changes"
```

### 9. **Push Your Changes**

```bash 
git push origin feature/your-feature-name

```

### 10. **Submit a Pull Request**

Go to the original WTTP repository and open a pull request (PR):

1. Navigate to your forked repository on GitHub.
2. Click Compare & pull request.
3. Provide a clear and detailed description of the changes in your PR.
4. Submit the pull request.

## **Code of Conduct**

By contributing, you agree to follow the WTTP Code of Conduct. Be respectful and professional when interacting with the community.

### Contribution Guidelines

Please ensure your contributions align with the following guidelines:

- Write clear and meaningful commit messages.
- Follow the WTTP Coding Standards.
- Test your changes thoroughly before submitting.
- Document any new functionality in the appropriate files.
